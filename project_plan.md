# 小学足球AI可视化反馈系统 - 核心规划与开发规范说明书

## 1. 项目基本信息与实验设计
* **论文题目**：AI可视化反馈赋能小学足球技能习得的路径机理与效果研究
* **研究对象**：小学五年级学生（10-11岁，正处于视觉控制向动觉控制过渡的敏感期）
* **研究场域与规模**：1所学校，共9个实验/对照班级（严格遵循整群随机对照试验 Cluster-RCT 设计）
* **技术载体**：计算机视觉（YOLOv8 + MediaPipe Pose） + 生成式人工智能（AIGC）
* **核心理论**：施密特指导假说（依赖效应体系）、必要难度理论（视觉-动觉错峰校准）、OPTIMAL理论（认知-动机双通道模型）

## 2. 核心生物力学诊断参数 (CK 映射)
针对儿童骨骼肌肉发育特征，系统聚焦于触球瞬间（t=0）的微观姿态解析，过滤动作冗余参数：
* **摆动腿触球瞬间膝关节屈曲角度（首要诊断指标）**：
    * **Green（达标）**：140° - 160°（专为儿童动作发展优化的黄金鞭打加速区间）
    * **Yellow（接近）**：130° - 140° 或 160° - 170°（偏离最优区间在 ±10° 以内）
    * **Red（错误）**：<130° 或 >170°（显著偏离，触发AIGC认知转译纠错提示）
* **支撑脚落位**：基于 YOLO 球心坐标与 MediaPipe 踝关节坐标，判定支撑脚尖是否位于球心外侧 15-20 cm 处。
* **髋关节旋转角度**：触球瞬间非支撑侧髋关节相对于支撑侧髋关节的水平旋转角度，最优区间为 25° - 40°。
* **踝关节状态**：触球瞬间角速度趋于稳定，判定为背屈锁定（二元分类：锁定/未锁定）。

## 3. 差异化教学路径与交互界面规范 (PK 映射)
系统必须严格分隔并支持以下两条独立的人机协同路径流，以验证“表现-学习悖论”：

### A组：实时反馈模式（高认知卸载 / 检验依赖效应）
* **触发机制**：YOLOv8 实时监控足球状态（静止 -> 瞬间位移），精确定位触球瞬间 $t=0$。
* **时序控制**：动作完成后静默 3 - 5 秒（保护即时本体感觉残留），随后自动触发反馈。
* **交互呈现**：0.5 倍速关键帧慢动作回放 + 半透明骨骼线叠加 + 目标关节红/黄/绿发光容错框 + AIGC 一句话积极提示文本。
* **生命周期**：界面呈现 8 - 12 秒后自动清空，切回摄像头实时预览画面，控制视觉通道不饱和。
* **教师规则**：课中核心练习（20分钟）严格执行“结构性沉默”，让渡纠错权给 AI，仅提供情感性社会支持。

### B组：延时反馈模式（适度认知卸载 / 验证必要难度）
* **课中状态**：核心练习期完全撤屏（黑屏或显示“数据采集中”），阻断实时视觉信息，迫使大脑调用内部错误检测机制。
* **后台逻辑**：系统静默捕获每次射门的关节坐标与角度数据，以匿名ID形式存入本地 JSON 日志。
* **离线反思报告（次课前5分钟结构化反思）**：
    * *组件一：双轴雷达图*：集中展示学生单次课所有射门尝试的膝关节与髋关节参数离散趋势。
    * *组件二：关键帧对比*：自动提取学生本次课绿框最多的一次动作，与内置标准模型并置左右分屏对比。
    * *组件三：个人进步曲线*：纵向呈现多时点错误检测能力（$d'$ 辨别力指数）的成长轨迹。
* **教师规则**：扮演“数据引导者”，利用启发式提问（如“回忆一下，踢得最顺的那脚身体是什么感觉？”）促进视觉-动觉的跨通道时延映射。

## 4. AIGC 积极意图认知转译引擎规范
* **底层驱动**：GLM-4 / 智谱AI系列大语言模型。
* **参数控制**：`temperature=0.3`（严格限制低随机性，确保教学法输出的一致性与安全性，消除大模型幻觉）。
* **转译三原则**：
    1.  *积极意图原则*：严禁出现评判性或惩罚性字眼（如“做得不对”、“动作错误”），必须使用信息性、建议性表述（如“下次尝试…”、“还可以做得更好”），保护感知胜任感。
    2.  *具身隐喻原则*：禁用抽象力学专业术语（如“角动量”、“屈曲度”），必须转译为儿童熟悉的身体隐喻（如“像大树的根稳稳扎地”、“腿部像拉满的弓弦”）。
    3.  *单一焦点原则*：每次反馈信息仅聚焦于当前输入中最严重的一个生物力学参数，防止小学生工作记忆超载。

## 5. 科技伦理与隐私保护红线
* **全边缘计算架构**：所有视频采集、YOLOv8 + MediaPipe 推理及画面渲染均在本地设备即时内存（RAM）中完成。严禁视频数据上传云端，严禁在本地磁盘持久化保存包含学生真实面部的原始 MP4 视频。
* **面部绝对脱敏**：在视频回放或渲染前，提取 MediaPipe 面部关键点（0-10）坐标区域向外扩展 10%，强制执行高斯模糊（`cv2.GaussianBlur`, 核尺寸 $21 \times 21$），确保面部特征完全不可辨识。
* **结构化落盘**：本地数据库或日志仅存储匿名化结构文本（如 `Subject_B004_Knee: 142`），割裂生物力学数据与真实个人身份的关联链。

## 6. 五阶段开发蓝图与里程碑
* **v0.1 基础视觉捕捉**：跑通本地摄像头，调用 MediaPipe Pose 实时绘制 33 点火柴人骨架，建立基础开发环境。
* **v0.2 体育力学诊断引擎**：编写向量数学计算模块，触球瞬间输出膝关节角度并渲染红/黄/绿三色容错框。
* **v0.3 AIGC 认知转译接入**：编写 API 代理模块，实现低随机性（`temperature=0.3`）的具身隐喻自然语言生成与主画面叠加。
* **v0.4 差异化双界面开发**：使用 PyQt5/PySide6 编写图形界面，彻底合拢 A 组（时序控制慢回放）与 B 组（黑屏采集 + 本地 JSON 日志）的独立交互流。
* **v1.0 完整科研量产版**：引入 YOLOv8 联动触发机制、面部实时高斯模糊、以及 B 组离线数据报告生成器（双轴雷达图与折线图自动化渲染）。
